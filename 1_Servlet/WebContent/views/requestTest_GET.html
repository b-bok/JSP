GET 방식으로 요청 후 응답페이지 보기

특징1. GET방식으로 요청하는 건 URL Header 영역에 데이터들을 포함시켜서 요청함
=> 사용자가 입력한 값(데이터)들이 url에 노출됨
=> 보안유지 불가능
=> 즉, 로그인 폼이나 회원가입 폼 같은 경우 GET방식에 부적합 함

특징2. Header에 데이터가 들어가기 때문에 전송하는 길이에 제한이 있음 => 초과된 데이터는 절단됨(즉, 짤림)
=> 즉, 게시판 작성하기 폼 같은 경우 GET방식에 부적합

특징3. 장점이라고 한다면, URL에 데이터가 포함되어있어, 즐겨찾기(북마크) 가능!(즐겨찾기 해놓고 다시 url 요청 가능)
=> 즉, 검색하기 폼 같은 경우 GET방식에 적합
