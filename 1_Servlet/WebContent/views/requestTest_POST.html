POST 방식으로 페이지 요청해보기

특징1. POST방식은 URL의 Body 영역에 데이터를 포함시켜 요청
=> 사용자가 입력한 값(데이터)들이 url 노출이 안됨
=> 보안유지가 가능함
=> 즉, 로그인 폼이나 회원가입 폼 같은 경우 POST방식에 적합함
특징2. 전송하는 길이에 제한 없음
=> 게시판 작성하기 폼 같은 경우 POST방식에 적합 함
특징3. 즐겨찾기 같은게 가능하나, 전달값이 url에 포함되어 있지 않아
=> 기존에 봤던 응답화면 볼 수 없을 수 있음

특징4. 최대 요청 받는 시간(TimeOut)이 존재해 페이지 요청, 기다리는 시간 있음!
